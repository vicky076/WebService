==========================================================================
    Visual Studio Team System：撰寫和執行測試概觀
==========================================================================

此概觀描述在 Visual Studio Team System 和 Visual Studio Team Edition for Software Testers 中撰寫和執行測試的功能。

開啟測試
-------------
若要開啟測試，請開啟內含測試定義的測試專案或測試中繼資料檔 (副檔名為 .vsmdi 的檔案)。您可以在 [方案總管] 中尋找測試專案和中繼資料檔。

檢視測試
-------------
若要查看可用的測試，請開啟 [測試檢視] 視窗。或者，如果您已安裝 Team Edition for Software Testers，也可以開啟 [測試清單編輯器] 視窗來檢視測試。

若要開啟 [測試檢視] 視窗，請按一下 [測試] 功能表，指向 [視窗]，然後按一下 [測試檢視]。若要開啟 [測試清單編輯器] 視窗 (如果您已安裝 Team Edition for Software Testers 的話)，請按一下 [測試]，指向 [視窗]，然後按一下 [測試清單編輯器]。

執行測試
-------------
您可以從 [測試檢視] 視窗和 [測試清單編輯器] 視窗執行測試。
請參閱＜檢視測試＞學習如何開啟這些視窗。若要執行 [測試檢視] 視窗中所顯示的一個或多個測試，請先在該視窗中選取測試。若要選取多個測試，請在點選測試時同時按住 Shift 或 CTRL 鍵。然後，按一下 [測試檢視] 視窗工具列中的 [執行測試] 按鈕。

如果您已安裝了 Visual Studio Team Edition for Software Testers，也可以使用 [測試清單編輯器] 來執行測試。若要在 [測試清單編輯器] 中執行測試，請選取您要執行之每個測試旁邊的核取方塊。然後，按一下 [測試清單編輯器] 視窗工具列中的 [執行測試] 按鈕。

檢視測試結果
--------------------
在您執行一個或一系列的測試時，測試回合的結果會顯示在 [測試結果] 視窗中。回合中的每個個別測試會顯示在單獨一行上，讓您可以查看它的狀態。在視窗的上半部會包含一個內嵌狀態列，提供您完整測試回合的摘要詳細資料。

若要查看特定測試結果的詳細結果，請在 [測試結果] 視窗中按兩下它。這麼做會開啟一個視窗，提供特定測試結果的詳細資訊，例如該測試所傳回的任何特定錯誤訊息。

變更測試的執行方式
-----------------------------------
每次執行一個或多個測試時，會使用設定集合來決定該些測試的執行方式。這些設定會包含在「測試回合組態」檔中。

以下是您可對測試回合組態檔進行之變更的部分清單： 

 - 變更每個測試回合的命名配置。
 - 變更在其上執行測試的測試控制器，讓您可以在遠端執行測試。
 - 蒐集所測試之程式碼的程式碼涵蓋範圍資料，讓您可以檢視您的測試所涵蓋的程式碼行數。
 - 啟用和停用測試部署。
 - 在執行測試前指定其他要部署的檔案。
 - 選取不同的主機 (ASP.NET) 執行 ASP.NET 單元測試。
 - 選取不同的主機 (智慧型裝置測試主機) 執行智慧型裝置單元測試。
 - 為執行測試的測試代理程式設定不同的屬性。
 - 在每個測試回合的開始和結束處執行自訂指令碼，讓您可以確實依每次執行測試的需要來設定測試環境。
 - 設定測試和測試回合的時間限制。
 - 設定在測試回合中的瀏覽器混合，和 Web 測試要重複的次數。

根據預設，每當建立新的測試專案時，同時會建立測試回合組態檔。您可以在 [方案總管] 中按兩下這個檔案，然後變更其設定，即可對此檔案進行變更。(測試回合組態檔會具有 .testrunconfig 的副檔名)。

一個方案可以包含多個測試回合組態檔。只會使用其中一個檔案 (稱為「現用的」測試回合組態檔) 來決定測試回合目前所使用的設定。您可以按一下 [測試] 功能表上的 [選取現用測試回合組態] 來選取現用的測試回合組態。

測試類型
----------
您可以使用 Visual Studio Team Edition for Software Testers 來建立數個不同的測試類型：

單元測試：使用單元測試，在會使用原始程式碼的C++、Visual C# 或 Visual Basic 中建立程式設計測試。單元測試會呼叫類別的方法、傳遞適當的參數，並驗證傳回的值是否為您所預期的值。
有三種特定的單元測試： 
 - 資料驅動單元測試，是當單元測試設定為要為資料來源的每個資料列重複呼叫時所產生的。單元測試會將每個資料列中的資料當做輸入資料。
 - ASP.NET 單元測試，是指在 ASP.NET Web 應用程式中使用程式碼的單元測試。
 - 智慧型裝置單元測試是部署至智慧型裝置或模擬器上，再由智慧型裝置測試主機執行的單元測試。

Web 測試：Web 測試是由一系列使用 Microsoft Internet Explorer 在瀏覽器工作階段 (Session) 中記錄之已排序的 HTTP 要求所組成。您可以具有關於測試所要求之網頁或網站的測試報告詳細資料，例如特定網頁是否包含指定的字串。

負載測試：您使用負載測試來封裝非手動的測試 (例如單元測試、Web 測試和一般測試)，然後透過虛擬使用者同時執行這些測試。在負載的情況下執行這些測試，便會產生測試結果 (包括效能和其他計數器)，而這些結果會以表格和圖形來表示。

一般測試：一般測試是在 Visual Studio 中包裝做為測試之用的現有程式。下列範例說明可以轉換成一般測試的測試或程式範例：
 - 現有測試，會使用處理序結束代碼以傳達測試通過或失敗。0 表示測試通過，而任何其他值表示失敗。
 - 在測試案例期間用於取得特定功能的一般程式。
 - 測試或程式，會使用特殊的 XML 檔案 (稱為「摘要結果檔案」) 以傳達詳細的結果。

手動測試：當您需要由測試工程師 (而非由自動化的指令碼) 完成測試工作時，便會使用手動測試類型。

已排序的測試：使用已排序的測試，依照您指定的順序執行一組測試。
